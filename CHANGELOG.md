# 更新日志

## v1.0.5 — 2026-07-12

本版集中强化“PPT 版 git”的可靠性、可诊断性和长期运行效率。

### 可靠性

- 保存只从同一份稳定临时副本完成解析、哈希和入库，规避 PowerPoint 原子保存竞态。
- 无效/截断 PPTX 不再写成恢复点；watcher 会按退避节奏补拍最终稳定版本。
- 去重快照入库前同时验证“可解析 + 规范内容哈希一致”，对象静默漂移时自动回退为完整快照。
- 恢复与导出改为同目录临时重组、PPTX 解析、规范内容哈希比对、原子替换四道门禁。
- 恢复前自动留当前版时会保护用户已选中的旧恢复点，避免恰逢配额上限时先清理、后恢复失败。
- 删除事件实时同步版本状态；离线巡检按持久游标轮转，不再长期饿死排序靠后的文档。
- 同路径离线重建会重新激活原文档；找回会跳过坏的最新版并退到最近健康版本。

### 完整性与维护

- 新增版本健康字段、快速仓库检查与用户可触发的全对象深度哈希检查。
- 损坏对象会映射到全部关联恢复点并隔离；浅检不会误清除深检隔离状态。
- 隔离版本不能恢复、导出、预览、从历史搜索入口操作，也不能作为复制文件的分支基线。
- “已留版 / 可回退”KPI 只统计健康恢复点；总版本与隔离版本保持独立物理口径。
- 清单与数据库规范内容哈希互校，发现元数据漂移时同样隔离。
- 手动深检与对象迁移/GC 使用独立可重入协调锁，避免迁移窗口造成假隔离，同时不阻塞保存快照。
- 重型对象迁移与 GC 改为七天节流；GC 保留分支恢复根，并在删除前验证完整引用图。
- 生产规模快速审计消除重复对象路径探测，实测由 109 秒降至约 2–3 秒。

### 体验

- 版本窗口新增按文件名/路径筛选，以及现存/已删除范围切换。
- 时间线明确标识并禁用无效恢复点；跨版本搜索沿用同一健康门禁。
- 设置页新增 50 / 100 / 200 / 不限保留上限与“深度检查版本库”。
- 默认保留 100 版，并优先保留跨编辑会话里程碑，避免一次密集保存挤掉长期历史。
- 隔离版本保留待修复但不占健康版本配额，不会挤掉可用恢复点。
- 开机自启快捷方式每次启动自愈到当前版本目录，修复升级后仍指向已删除旧 EXE 的问题。
