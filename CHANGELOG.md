# 更新日志

## v1.1.0 — 2026-07-16

界面全面换代为「静白工作室」设计语言：像成熟桌面软件一样安静、精确，信息主次分明。

### 新默认主题：静白 / 静黑

- 不透明纯净层级取代半透明玻璃与极光光团；深色态背景不再发灰发脏，文字对比稳定。
- 旧 10 套极光玻璃主题全部保留，可在主题菜单随时切回；已选旧主题的用户不受影响。
- 强调色预算制：整屏蓝色只允许出现在选中指示条、主按钮与焦点环；命中词高亮改为荧光笔黄。

### 布局与控件

- 标题栏与搜索栏合一：品牌、搜索框、类型 / 范围下拉、图标按钮、窗口控制同居一行，顶部省出一整行高度；标题栏空白处仍可拖动窗口，交互控件处不误触拖拽。
- 筛选 / 设置 / 主题 / 胶片报告四个文字按钮改为 32px 图标按钮，悬停出完整说明。
- 结果卡片：命中页码合并为按页码排序的等宽灰标签；匹配类型徽章灰化；文件名悬停显示全名；修改时间等宽对齐。
- 预览区：路径面包屑化（目录淡化、文件名加粗，悬停看完整路径）；页码按钮变分段控件并按页码排序；头部动作收敛为「复制本页文字 / 复制路径 / 详情」一行；底部仅保留一个实心主按钮。
- 渲染失败文案从「COM 原图渲染失败」改为「暂时无法生成原图预览」+ 可执行的重试指引；主按钮不可用时正确置灰。

### 一致性细节

- 控件圆角三档制（卡片 12 / 控件 8 / 小件 5），hover 只变底色，Tab 焦点环全局统一为一根强调色描边。
- 数字、路径、耗时、热键统一等宽字体；状态栏热键标签键帽化。

## v1.0.15 — 2026-07-16

本版解决 v1.0.14 的核心可用性缺口：用户正在使用 PowerPoint 时，PPT Doctor 不再直接拒绝预览；同时不冒充“多实例隔离”，而是用可审计的隐藏只读快照安全共用单实例 COM。

### 用户 PowerPoint 打开时也能看 COM 原图

- 没有 PowerPoint 时，继续创建一个隐藏实例，并以新 PID、COM/DcomLaunch 创建父进程、精确进程句柄、无可见窗口四重证据确认它属于 PPT Doctor；用户恰好同时手动启动 Office 时不会把该 PID 误认成自有进程。
- 用户 PowerPoint 已打开时，前台预览只借用其 `Application` 引用，实际打开的是缓存目录中的 `ReadOnly=1, WithWindow=0` 快照；不碰原文件，不修改 `Visible`、活动文稿、显示设置或保存状态。
- 借用后强制审计：快照必须只读、窗口数为 0、路径精确匹配，用户窗口总数和活动文稿必须保持不变；任一条件不能证明即关闭快照并失败，不扩大权限。
- 清理只关闭本工具持有的快照。借用模式永不调用 `Application.Quit`、永不终止用户 PowerPoint PID；临时关闭被 COM 拒绝时保留精确对象引用继续重试，避免随机名文稿遗留任务栏。
- 失败提示不再错误要求用户关闭 PowerPoint；若 Office 正忙或有弹窗，只需完成当前操作后重试。

### 单通道更快，而不是堆 COM 客户端

- Microsoft 将 PowerPoint 定义为 Multiuse（Single Instance）COM 服务；本机反复实测 `DispatchEx` 的 2–3 个客户端始终共用一个 `POWERPNT.EXE`。
- 固定导出 12 页时，单客户端约 628 ms；双客户端约 721 ms；三客户端复测约 668 ms、另一轮约 866 ms。多客户端没有吞吐收益，反而增加争用、CPU 和内存，因此继续保持一个串行 COM 通道。
- 用户 PowerPoint 已打开时，真实产品路径首张 1600px 预览约 251–315 ms；复用同一隐藏快照导出相邻页约 49–116 ms。渲染前后用户窗口、活动文稿和 Saved 状态均不变。
- 空闲 350 ms 后仍完整释放快照和 COM，但保留隔离渲染服务进程阻塞在本地 socket 上。实测其空闲 CPU 为 0%、RSS 约 29 MB；下一次预览免去子进程重启与握手，约节省 262–615 ms。
- 没有用户 PowerPoint 时，冷启动仍由 Office 自身主导（本机约 3.3 秒）；第二次在复用渲染服务进程后约 2.7 秒。每次空闲释放后隐藏 `POWERPNT.EXE` 都会退出，不以长驻 Office 换速度。

### 验证

- 新增前台借用授权、后台禁借、只读无窗口审计、活动文稿不变、所有权降级、IPC 参数透传、COM 释放但子进程复用、最终退出等红绿回归。
- 真机 E2E 同时覆盖“用户可见文稿 + 产品直连渲染”“用户可见文稿 + IPC 渲染”“无用户 PowerPoint + 自有隐藏实例”三条路径；每轮结束均确认无多余窗口、无哈希文稿、无 PowerPoint / 渲染子进程残留。

## v1.0.14 — 2026-07-15

本版纠正 v1.0.13 把“可读内容”当成“页面预览”的产品偏差：右侧预览重新收敛为单一 PowerPoint COM 原图链路，不再提供文字、封面、Shell、PDF 或第三方渲染降级。

### COM 原图成为唯一结果

- 未命中合格缓存时，界面只显示“正在启动 / 等待 PowerPoint COM 原图渲染”；成功后显示 COM 直接导出的页面，失败则明确显示“COM 原图渲染失败”。
- 删除命中页文字速览 API、并行安全预览任务、LibreOffice 转 PDF、QtPdf 光栅化以及“带预览引擎完整版”打包路线，减少额外 CPU、进程和模糊的产品语义。
- 右侧只接受达到当前显示需求的 1920–2560 长边 COM PNG；不再把 960px 缓存放大成临时预览。相邻三页仍由同一个串行 COM 会话以 1920px 预取，翻页不增加并发。
- 缓存键加入 `com-only-v1` 代际，v1.0.13 可能由非 COM 路线生成的图片不会再被读取；缓存维护会主动清理文字速览、兼容 PDF、临时工作区与旧 profile。
- 若用户 PowerPoint 已在运行，仍拒绝把哈希快照塞入其单实例会话；此时明确失败并提示关闭 PowerPoint 后重试，绝不以降级图掩盖失败，也不重演随机文件名与显示状态污染。

### 任务栏图标

- EXE、Qt 窗口、托盘和桌面快捷方式统一使用同一份多尺寸 `app.ico`；快捷方式直接读取 EXE 第 0 个图标资源，并移除与普通 `.lnk` 不一致的自定义 AppUserModelID。

### 验证

- 新增严格 COM-only 红测：活动 PowerPoint 场景不得调用任何非 COM 路线、缓存代际必须与旧图隔离、文字预览 API 必须不存在。
- 冻结包不再携带 `Qt6Pdf.dll` / `QtPdf.pyd`，分发体积和运行依赖同步收口。

## v1.0.13 — 2026-07-15

本版继续收口“后台工作存在，但不能抢走前台体验”。重点优化全盘补漏时的状态呈现，以及用户已经打开 PowerPoint 时的预览安全与翻页速度。

### 扫描状态更安静

- 全盘扫描不再把长路径和持续变化的文件名塞满状态栏；界面只保留“扫描 / 建库”、发现或完成数量、百分比与预计剩余时间，详细路径移到悬浮提示。
- 新进度条只在真正扫描且可见时以 20 FPS 粗粒度动画；确定进度、隐藏窗口和任务结束后立即停表，减少无意义重绘与视觉噪声。
- 搜索进行中会继续保留搜索状态，不被后台建库文案覆盖；用户能清楚区分“我正在搜”与“软件空闲补库”。

### PowerPoint 已打开时也不污染会话

- 预览仍然禁止借用用户正在运行的 PowerPoint；即使能精确定位窗口，也不在其中打开临时快照，根除 `Close` 短暂失败后随机名演示文稿滞留任务栏的风险。
- 第 1 页优先读取 PPT 内嵌封面；任意后续命中页可在不启动 Office 的情况下生成明确标注的“文字速览”，只解析被点中的单页。高清图稍后安全替换，高清失败时速览仍保留，不再把核心能力直接降级成“无法预览”。
- 文字速览改走独立的“只留最新请求”交互通道，不再被前一个文件最长 30 秒的高清转换堵住；请求编号与高清完成标记双重防护，旧页或迟到的速览不会覆盖当前高清图。
- 旧版 `.ppt`、加密、结构异常或索引页码过期时，也会显示明确的安全说明图，不再只留空白失败页；原文件不会因此被读取写回或改动。
- 若本机有 LibreOffice，用户明确点击预览且 PowerPoint 正忙时，会用独立低优先级进程把安全副本一次性转为缓存 PDF；首次转换后，后续翻页只在本地渲染缓存页，不接触 PowerPoint。
- 修复 LibreOffice 已成功产出 PDF、但退出尾声临时目录仍被占用时被误判为“预览失败”的收尾竞态；清理失败不再抹掉已经成功的首帧结果。
- 兼容 PDF 纳入 2GB / 2000 项统一 LRU 上限，残留工作目录 10 分钟后回收；转换超时只终止命令行带本次随机隔离 profile 的 LibreOffice 进程，不会误杀用户自己的 LibreOffice。
- LibreOffice 兜底不用于后台预取，最多等待 30 秒且按文件内容缓存；缺少 LibreOffice 时仍保留内嵌封面、既有缓存或命中页文字速览。

### 验证

- 新增用户 PowerPoint 会话禁借、任意命中页速览、旧格式说明页、迟到速览防覆盖、LibreOffice 一次转换多页复用、缓存回收、临时目录清理竞态、精确超时进程回收等回归测试；冻结版自检真实执行 QtPdf 的 PDF→PNG 路径。
- 本机最终代码实测：未缓存的命中页文字速览约 442 毫秒；独立高清转换首次约 12.2 秒，缓存后翻到第 2 页约 100 毫秒；转换结束无残留 LibreOffice / PowerPoint 进程。
- 最终源码门禁 `899 passed, 2 deselected`、退出码 0；冻结版内容搜索自检除 `12/12` 搜索用例外，额外真实验证 QtPdf 的 PDF→PNG 路径，冻结 watcher 改存留版验证通过。

## v1.0.12 — 2026-07-15

本版把 v1.0.11 的六大主题胶片报告与新一轮深度流畅度治理合并交付。目标不是让后台任务跑得更快，而是让搜索、排序、翻页、打开文件始终优先，空闲时再低负载补齐全盘索引与版本保护。

### 默认轻量、按需增强

- 新用户默认进入基础模式：只开启 PPT 搜索与统计；PPT 自动版本管理、Word/PDF 搜索和智能分组均可在设置中独立开启，避免未使用的能力常驻消耗 CPU、I/O 与内存。
- 功能开关与数据状态分离：关闭增强能力不会删除已有索引或历史版本；重新开启后可继续使用，降低试用成本与误操作风险。
- 全盘补漏仍覆盖固定磁盘以及 `AppData\Roaming` 等企业软件目录，但自动扫描固定为低优先级、最多两个解析进程；检测到搜索、翻页、打开/恢复文件等用户交互时暂停继续派发后台任务。
- 扫描被取消、超时或磁盘暂时不可用时，不再把本轮未看到的文件当成已删除，防止不完整枚举污染索引。

### 搜索与界面不再互相拖累

- 搜索数据库改为只读、短超时连接；写库繁忙时快速返回并由界面合并重试，不再让“正在搜索”长期挂住。全文结果仍保留原文验证、完整短语优先、大小写开关和主/次排序组合。
- 排序、筛选、版本列表、胶片报告等文件与数据库读取全部移出 GUI 主线程；后台任务完成或取消后立即退役，修复长时间使用中线程对象与回调逐步堆积的问题。
- 搜索结果继续采用纯文本卡片，不生成封面缩略图；渲染资源集中给用户选中的右侧大图和实际翻页。
- 后台任务池为打开、恢复、导出等显式操作保留通道；体检、统计、扫描不能占满所有执行槽。

### PowerPoint 预览与安全边界

- 预览继续运行在隔离渲染进程中，并加强自有 PowerPoint PID、窗口句柄和临时演示文稿的有界清理；只处理本工具创建的隐藏实例，绝不关闭用户已有 PowerPoint。
- 翻页采用方向感知的三页小窗口缓存：当前页优先，随后只在同一安全会话空闲时预取前后相邻页；连续翻页会淘汰过时请求，不额外并发启动 PowerPoint。
- 渲染缓存按文件版本与目标清晰度复用；已有可用图先显示，再在用户停留后补高清，避免重复导出同一页。
- 版本恢复/导出若后端提交失败，界面会回滚忙碌状态并明确报错，不再留下按钮永久禁用或“处理中”假状态。

### 验证

- 新增基础/增强模式、低优先级全盘扫描、只读搜索退避、渲染 IPC 与缓存、任务退役、版本操作回滚等回归测试。
- 独立长时压力测试覆盖持续搜索、排序、预览与空闲观测；发布包继续执行真实冻结环境的 12 项内容搜索自检。
- 合并后交叉回归 `474 passed`，完整测试门禁 `881 passed`；两组均为零失败退出。

## v1.0.10 — 2026-07-14

本版继续围绕“任何后台工作都不能绑架用户操作”做深度收口，重点修复渲染阻塞、退出残留、后台任务饥饿和大版本库筛选卡顿。

### PowerPoint 打开与退出

- 渲染 IPC 增加独立于请求锁的紧急中断通道：用户打开原稿或退出时，可以直接关闭本工具的 socket 与隔离子进程，不再被最长 20 秒的 COM 请求锁反向卡住。
- 打开原稿先给预览 750 ms 收尾；仍未释放则中断隔离渲染器，并等待隐藏 `POWERPNT.EXE` 消失。若无法确认安全交接，拒绝打开并提示重试，绝不把原稿交给低清预览会话。
- 主窗口退出移除 `QThread.terminate()`；渲染线程采用 stop → abort → bounded wait，更新线程通过关闭活动 HTTP response 唤醒阻塞读，让 finally 清理可以正常执行。
- 历史版本预览新增 IPC `render_once`：渲染、关闭临时演示文稿在同一子进程请求内完成，不再调用全局 renderer shutdown 干扰当前预览。

### 交互优先与低负载

- 通用后台任务池为打开、恢复、导出等显式用户操作保留一个通道；普通诊断、历史查询最多占其余通道，避免按钮点击长期排队。
- 等待执行的后台任务支持取消，主窗口退出会先停止所有队列任务，再做有界等待。
- 渲染诊断改为非阻塞读取；渲染正忙时返回 `busy` 状态，不在 GUI 线程等待请求锁。
- 自动全盘补漏同时降低索引线程与解析子进程的 Windows CPU/I/O 优先级，并固定单解析进程。
- 修复自动单 worker 扫描走 inline、绕过 60 秒解析超时的问题；隔离进程不可用时自动任务安全中止，不回退到不可终止的线程池。

## v1.0.9 — 2026-07-14

本版修复预览自动化会话泄漏到用户 PowerPoint、导致画面持续模糊并在任务栏出现哈希文件名快照的核心缺陷，同时完成全盘补漏、结果卡降载、翻页快切和胶片报告增强。

### PowerPoint 会话隔离

- 根因确认：旧“打开并跳页”直接用 COM 打开原稿，可能复用预览渲染器创建的隐藏 PowerPoint 进程；工具快照与用户原稿因此共处一个自动化会话，退出 PPT Doctor 后该 PowerPoint 进程仍会继续影响显示。

- 无文件变化的覆盖扫描跳过全库版本聚类和 SQLite 维护，保留完整枚举能力而不制造无意义 CPU/IO。

- 搜索结果卡改为纯文本布局，移除封面占位图、可见区滚动定时器、缩略图 worker、请求优先队列和 256 张内存缓存；搜索、排序、滚动、展开版本组不再触发任何结果图片工作。
- 图片资源只服务于用户实际选中的文件：右侧大图、命中页导航和最多一张相邻页预取；完成的大图不再复制回结果卡。
- 胶片报告新增“创作足迹”：活跃修改日、最长连续活跃天数、最忙月份和当前库修改时间跨度，明确按文件最后修改时间统计。
- 胶片报告新增“胶片 DNA”：平均页数、每页文字密度、短篇/长篇分布、近似内容复用家族和正文索引就绪率。
