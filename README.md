## 🛡️ v1.0.15：PowerPoint 开着也能安全预览

- 用户正在编辑 PPT 时，右侧预览可直接借用现有 PowerPoint COM，但只打开本地只读、无窗口的临时快照；原文件、活动文稿、保存状态和显示设置都不改。
- 每次借用都核验快照只读、窗口数为 0、路径匹配、用户窗口数和活动文稿不变；任一证据不成立即关闭快照并失败，绝不 `Quit` 或终止用户的 PowerPoint。
- 没有 PowerPoint 时仍使用 PPT Doctor 自己创建并验证 PID 与 COM 创建父进程的隐藏实例；空闲后立即释放 Office，不让隐藏实例被用户下一次打开复用。
- PowerPoint 官方 COM 是单实例模型。本机两轮实测 2–3 个 COM 客户端没有加速、反而增加争用，因此坚持单通道，把预算用于当前页和相邻页缓存。
- 渲染子进程与 PowerPoint 分开释放：Office 空闲即退出，29 MB 的隔离服务进程保持零 CPU 休眠。用户 PowerPoint 已打开时首张实测约 0.25–0.63 秒，相邻页约 0.05–0.12 秒。
- 渲染失败时不再要求关闭 PowerPoint；若 Office 正忙或有弹窗，处理完后重新点击即可。

## 🖼️ v1.0.14：只显示 PowerPoint COM 原图

- 右侧预览只有三种状态：PowerPoint COM 原图渲染中、原图成功、明确失败；不再用文字、封面、Shell、PDF 或第三方渲染结果代替原页。
- 只读取 1920–2560 长边的 COM 直接导出图，不再先把低清缓存放大；同一串行 COM 会话会以 1920px 预取相邻三页，兼顾清晰度与翻页速度。
- 缓存切换到 `com-only-v1` 新代际，旧版可能由非 COM 路线生成的图片全部失效；启动维护会清理文字速览、兼容 PDF 与临时工作区。
- 用户已经打开 PowerPoint 时仍不借用其单实例会话：明确提示关闭后重试，避免随机名临时稿、任务栏污染和显示分辨率异常复发。
- 窗口、托盘、EXE 和桌面快捷方式统一使用多尺寸 `app.ico`，并移除导致任务栏身份不一致的自定义 AppUserModelID。

## 🧊 v1.0.13：扫描不霸屏，预览不借用户 PowerPoint

- 全盘补漏改为安静的阶段/数量/百分比状态，详细路径只放悬浮提示；进度动画只在任务可见且总量未知时运行，结束或隐藏立即停表。
- 用户已经打开 PowerPoint 时，PPT Doctor 仍不会把任何临时稿塞进该会话；即使精确绑定窗口也不冒险，避免随机文件名与显示状态污染复发。
- 第 1 页先显示 PPT 内嵌封面；任意后续命中页通过独立交互通道生成明确标注的“文字速览”，不受前一个高清转换阻塞。旧版、加密或异常文件也会给出安全说明图，不再直接掉到“无法预览”。
- 若本机安装 LibreOffice，明确点击后可在独立低优先级进程中生成缓存 PDF 并自动替换速览，后续翻页约 0.1–0.3 秒；没有该组件时仍保留封面、已有高清缓存或命中页文字速览。
- 兼容 PDF 与残留临时目录有统一空间上限和过期回收；超时只清理由本次随机隔离 profile 启动的 LibreOffice，不碰用户自己的 Office 会话。

## ⚡ v1.0.12：交互优先、默认轻量与预览快切

- 新用户默认只开启 PPT 搜索与统计；自动版本管理、Word/PDF 搜索、智能分组可独立启用。关闭能力不删已有数据，重新开启即可继续使用。
- 全盘补漏仍覆盖固定磁盘和 `AppData\Roaming`，但自动任务以低优先级、最多两个解析进程运行；搜索、翻页、打开或恢复文件时暂停继续派发后台工作。
- 搜索使用只读短超时连接，写库繁忙时快速让出并合并重试；排序、筛选、版本列表、报告读取都离开 GUI 主线程，“正在搜索”不再被后台 I/O 长时间拖住。
- 搜索结果保持纯文本卡片，不渲染封面；右侧预览复用文件版本/清晰度缓存，以当前页为最高优先级，并在安全空闲窗口预取翻页方向相邻页。
- 隔离渲染器只清理本工具拥有的隐藏 PowerPoint PID、窗口和临时稿，绝不关闭用户已有 PowerPoint；退出与打开原稿均采用有界收尾。
- 后台任务完成或取消后立即退役，避免长时间使用时线程、回调和句柄累积；版本恢复/导出失败也会完整回滚界面状态。

## 🧪 v1.0.10：卡死熔断、干净退出与交互优先

- 打开原稿不再被正在渲染的 PowerPoint COM 请求拖住：先给预览 750 ms 温和收尾，仍卡住就只中断本工具的隔离渲染子进程；确认隐藏预览进程退出后才交给 Windows 打开，杜绝再次进入低清自动化会话。
- 退出软件不再强杀 Qt 渲染/更新线程；会主动关闭隔离渲染通道和网络响应，让清理代码真正执行，避免后台残留 PowerPoint、随机名临时演示文稿或下次启动异常。
- 历史版本预览改为子进程内“渲染一页后立即关闭”的原子操作，不再关闭主预览会话；看完旧版本后继续翻当前 PPT，不必重新冷启动 PowerPoint。
- 后台任务为“打开/恢复/导出”保留一个交互通道，体检、统计等慢任务占满时，用户点击也不会一直排队；退出时尚未开始的任务可取消。
- 自动全盘补漏固定使用一个低优先级、可终止的隔离进程；坏文件超过 60 秒会跳过，隔离进程无法启动时安全中止，不会退化成无法杀掉的线程长期占 CPU。
- 版本管理的几千文档筛选改为每轮最多构建 160 行，切换“全部/现存/已删除”不再一次性堵塞界面。
- 设置诊断读取渲染状态改为非阻塞快照，预览正忙时打开设置不再跟着最长 20 秒超时一起卡；更新下载退出也改为关闭活动连接、协作收尾。

- 修复使用 PPT Doctor 打开原稿后画面变糊、任务栏出现哈希/随机文件名 PPT 的会话污染：打开前彻底释放工具预览会话，原稿改由 Windows 正常打开，COM 只附着已打开文档执行跳页。

- 已有索引升级后自动安排一次低优先级补漏，此后每 7 天在空闲窗口完整枚举固定磁盘；自动任务固定单 worker，搜索或文件操作进行中自动延期，无变化时不做全库聚类和数据库维护。
- 右侧预览只预取当前翻页方向的一页，已有 960px 缓存先瞬时显示；大窗口停留 160ms 后才补高清，连续翻页会取消未开始的高清任务，已达目标清晰度的缓存不再重复调用 PowerPoint。
- 搜索结果改为紧凑纯文本卡片，不再加载每一项的封面图；搜索、排序和滚动产生 0 个结果缩略图任务，把图片解码与渲染预算留给用户点击后的右侧大图和相邻页。
- 胶片报告增加“创作足迹”与“胶片 DNA”：连续活跃期、最忙月份、平均页数、文字密度、短篇/长篇、复用家族和正文索引健康度；范围切换在 SQL 层先过滤并复用已生成报告。

- 📊 **库概览仪表盘**：KPI + 主题分布 / 最近活跃 / 库构成 / 本周改动趋势，打开即见全景。
- 🎨 **静白工作室 UI**：默认「静白 / 静黑」双主题 + 10 套极光玻璃备选，合一工具栏 + 左侧导航轨单窗工作台。
- ⚡ **托盘常驻 + 全局热键**：`Alt+F` 随时唤起。

- 多词搜索先按完整短语排序：`AI SP` 优先命中连续的 `AI SP`，同时保留 `AI` 与 `SP` 同页分开出现的召回兜底。
- 新增“大小写”开关，默认不区分；开启后文件名与页内容都按原文大小写验证。

- 翻页预取只复用本工具已打开的安全快照，绝不为后台任务启动/附着 PowerPoint 或打开新演示文稿；单 COM 串行、最多两页、120 ms 可取消空闲窗。
- 优先预取当前翻页方向的相邻页和下一命中页；右侧高清预览完成后直接复用为当前结果卡缩略图，不增加一次 COM 渲染。

## ⚡ v1.0.7 流畅度升级

- 相关度按“文件名完全匹配 > 内容完整匹配 > 部分匹配”排序，并支持主排序 + 次排序组合。
- 大结果集只创建首屏卡片，其余按需加载；排序和筛选不再反复重建数百个 QWidget。
- 搜索状态与索引统计解耦，文件变化引发的重复搜索合并到空闲窗口。
- 预览复用安全快照、只预取一页；超时不重复等待，后台不再启动时自动预热 PowerPoint。
- 非空索引启动时不再周期性遍历全盘；只核对已索引路径及其直接父目录，避免启动后长期占满一个 CPU 核。关闭期间在全新目录创建的文件、或离线删除的文件，可用“重新扫描全盘”同步。
- 首次建库若被中断，待解析文件会在下次运行中按节流队列持续补齐，无需反复重启；需要立即补全时仍可手动重新扫描。

## 🚀 用法

绿色免安装，双击 `PPT Doctor.exe` 即用。首次自动后台全盘建索引（边扫边搜）。

源码运行：

```bash
uv run python -m pptx_finder
```

打包：

```bash
uv run pyinstaller pptx-finder.spec --noconfirm    # → dist/PPT Doctor/PPT Doctor.exe
```

## 🛡️ v1.0.5 版本守护

- 保存事件先捕获为稳定临时副本；PowerPoint 原子替换或同步盘改写期间，不会把两次保存混成一个版本。
- 每个恢复点都是完整 OpenXML 包快照：跨文档对象去重，但恢复不依赖补丁链。
- 恢复/导出先在目标同目录重组、解析并核对内容哈希，全部通过后才原子替换；失败不动现有文件。
- 启动快速检查清单和对象引用，设置页可手动执行全对象哈希深检；损坏恢复点自动隔离，不再参与恢复、搜索入口或复制分支。
- 离线巡检按游标轮转覆盖全部受管文档，并能重新接管软件关闭期间在原路径重建的文件。
- 默认每份 PPT 保留 100 版：先保每次编辑会话的里程碑，再保近期细粒度保存；复制分支的基线永久受保护。
- 重型迁移与 GC 七天节流；生产规模快速审计由重复磁盘探测改为一次对象集合扫描。
