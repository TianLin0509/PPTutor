# pptx-finder · 本地单文件版本管理 设计规格（spec）

> 让 PPT 像 git 一样有版本，但用户**零操作、无感**。桌面永远一个干净文件，历史自动留在后台，丢不了、乱不了、找得到。
> 本文件是开发契约。模块按此实现，避免漂移。与现有 `docs/design.md`（搜索/预览）并列，复用其能力。
> 状态：已与用户确认方向与功能取舍（2026-06-19）。

---

## 0. 灵魂原则：零操作负担

**目标用户大多没有版本控制习惯。任何需要用户主动操作的功能都是负担，能砍则砍。**

- ✅ 一切**自动发生**：纳管一次之后，用户只管照常用 PowerPoint、Ctrl+S 保存，其余（拍快照、折叠、缩略图、找回的底）全在后台无感完成。
- ✅ 唯一的「用户动作」是低频的：恢复/导出（出事那天才点一次）、首次设置受管目录。
- ❌ 凡是要求用户「主动打标签、写备注、手动提交」的——**一律不做**（已砍：里程碑标签、版本备注）。
- ✅ 纯本地，不上传、不依赖云/账号（内网/保密友好）。
- ✅ 不动用户原文件——版本库独立存放，只读不改原 pptx。

## 1. 范围与优先级（已确认）

### 做（按阶段落地）
- **单线版本管理**：文件夹级纳管 → 监听保存 → 每次「保存且内容变」自动记一版 → 版本时间线（预览/恢复/导出）→ 离线启动补记。
- **统一设置面板**：收纳 风格 / 全局热键 / 开机自启 / 受管目录 / 版本库配额。
- **开机自启**：随 Windows 启动到托盘（规避「忘了开」漏版本）。
- **按页去重存储**：pptx 拆 part 内容寻址去重，改几个字只多几十 KB。
- **细碎版本自动折叠 + 配额清理**：一次编辑会话的多次保存自动折叠为一组；老版本按配额自动归并/清理。**全自动，无需用户操作。**
- **每版自动缩略图**：时间线每版配一张图，扫一眼认出。
- **误删 / 改坏一键找回**：原文件被删/存坏，从版本库重建完整 pptx。
- **跨版本内容搜索**：搜「以前某版有、现在删了的内容」，一搜直达那个历史版本。**finder 独有护城河**（复用全文索引）。

### 砍（明确不做）
- ❌ 里程碑标签 / 版本备注（需用户主动操作，违背零负担原则）
- ❌ 手动「打快照」按钮、页级溯源、版本库迁移导出、（面向用户的）恢复自检入口（第三档）

### 非目标（本 spec 不涉及，留未来）
- ☁️ 云同步 / 跨设备
- 🌿 分支与合并（协作分叉的三方 merge）——见 §12 未来扩展
- 🤝 协作 diff / 变更报告 / 版本包跨人传递——见 §12
- 错字模糊搜、OCR 图片字

## 2. 核心机制：完整快照 + 按页去重

**回退绝对可靠的根基：每个版本都是「那一刻的完整快照」，不是补丁链。**

- pptx 本质是 zip，内部是多个 part（`slide1.xml`、`media/*.png`、`presentation.xml` 等）。
- 每次保存：解压所有 part → 每个 part 算 `xxhash` → 新内容存进对象库（重复的不存）→ 写一份 manifest（这一版 = 哪些 part 的 hash + 页顺序 + 每页文本）。
- **回退/找回 = 按某版 manifest，从对象库取回各 part，重新打包成 pptx。** 每版独立自包含，与其他版本无关，永不断链。
- 删页/加页/大改 → 每版都精确捕获那一刻完整状态（被删页的内容仍存在删除前那版的对象里）。

> diff（「这版改了哪几页」）是**比对两份完整快照**算出来的展示层能力，与回退（取快照）解耦。

## 3. 用户体验

| 场景 | 用户做什么 | finder 自动做什么 |
|------|-----------|------------------|
| 纳管 | 在设置里指定工作目录（一次性） | 该目录下所有 .pptx（含新建）纳入管理 |
| 日常编辑 | 照常 PowerPoint 改、Ctrl+S | 检测保存且内容变 → 后台拍快照（去重，无感） |
| 看历史 | 点开某文档 | 展示版本时间线（时间 + 自动「改了第几页」+ 缩略图），会话折叠 |
| 回退 | 选某版 → 恢复 / 导出 | 重组该版完整 pptx（恢复前自动再存当前，不丢） |
| 对外发版 | 选某版 → 导出 | 生成独立规范命名文件（平时不产生散落 v 文件） |
| 误删/改坏 | 点「找回」 | 从版本库重建出完整文件 |
| 找回流内容 | 搜一个词，勾「含历史版本」 | 在所有历史版本里全文检索，命中直达「某文档·某版·某页」 |

界面：在主窗（或预览区）为「受管文档」增加**版本时间线**面板；设置入口为顶栏统一「设置」按钮。

## 4. 存储设计（版本库 vault）

位置：`%LOCALAPPDATA%\pptx-finder\vault\`（与 index.db 同级，纯本地，独立于用户原文件）。

```
vault/
  <doc_id>/                      doc_id = 规范化原路径的 hash（身份，见 §7）
    objects/<part_hash>          去重的 part 内容块（slide XML / 图片 / …）
    versions/<version_id>.json   该版 manifest（见 §6）
    thumbs/<version_id>.png      该版缩略图
    meta.json                    原路径 / 状态(active|deleted) / 当前版本
```

MVP 阶段可先用「完整 pptx 拷贝」（`versions/<id>.pptx`）跑通最可靠，阶段 2 再切换到 `objects/` 去重。

## 5. 技术栈 / 依赖（全部已具备）

- 监听：`watchdog`（已依赖）—— 监听受管目录写入事件 + 防抖
- 拆包/重组：`zipfile`（标准库）—— pptx = zip
- 内容寻址：`xxhash`（已依赖）—— part 去重指纹
- 认亲/归组：`datasketch` MinHash-LSH（已依赖）—— 同源识别、跨版本关联
- 渲染缩略图：`win32com`（已依赖）—— 复用 renderer
- 索引：SQLite FTS5（标准库）—— 跨版本内容搜索复用
- 开机自启：注册表 `HKCU\...\Run` 或启动文件夹快捷方式

**无新增重依赖。**

## 6. 数据模型

### 6.1 SQLite 新增表（与现有库同库）

```sql
CREATE TABLE managed_roots(            -- 受管文件夹
  path TEXT PRIMARY KEY, added_at REAL
);
CREATE TABLE managed_docs(             -- 受管文档
  doc_id TEXT PRIMARY KEY,             -- 规范化路径 hash
  path TEXT,                           -- 当前原文件路径
  status TEXT,                         -- active | deleted（原文件删了，vault 保留）
  latest_version_id TEXT,
  created_at REAL, updated_at REAL
);
CREATE TABLE versions(                 -- 版本记录
  version_id TEXT PRIMARY KEY,         -- 时间戳派生 id
  doc_id TEXT, ts REAL,
  session_id TEXT,                     -- 编辑会话（折叠用：间隔 < 阈值同一会话）
  page_count INTEGER, size INTEGER,
  changed TEXT,                        -- 自动算的「改了第几页」摘要
  thumb_path TEXT, manifest_path TEXT
);
-- 跨版本内容搜索：历史版本逐页文本入 FTS
CREATE VIRTUAL TABLE version_pages_fts USING fts5(
  content, doc_id UNINDEXED, version_id UNINDEXED, page_no UNINDEXED
);
```

### 6.2 manifest（`versions/<id>.json`）

```
{ "ts": <unix>, "page_count": 20,
  "slide_order": ["rId2","rId3",...],          // presentation.xml 的页顺序
  "parts": { "ppt/slides/slide1.xml": "<hash>", "ppt/media/image1.png": "<hash>", ... },
  "page_texts": [ {"page_no":1,"text":"..."}, ... ]   // 供跨版本搜索 / diff
}
```

## 7. 关键流程

- **文档身份 doc_id**：以规范化原文件路径 hash 为主键。「另存为」到新路径时，用 MinHash 与已有受管文档比对，**提示**是否并入原版本链（不自动并）。
- **监听与触发**：watchdog 监听 `managed_roots`；PowerPoint 原子保存（写临时文件→替换）会产生 created/moved，**防抖**（事件后等文件稳定且可读）→ 算内容指纹，**与最新版不同才记**（内容没变的保存跳过）。
- **离线补记**：启动时遍历 active 文档，比对原文件当前指纹 vs `latest_version` → 不同则补记一版（标「离线改动」）。中间态无法补（靠开机自启规避）。
- **会话折叠**：相邻保存间隔 < 阈值（默认 30 min）归为同一 `session_id`，时间线默认折叠为一组，可展开。
- **配额清理**：默认策略——近期会话全留；更老的每会话留 1 版（首/末）；超总量上限按 LRU 归并。全自动。
- **找回**：原文件被删 → managed_docs.status=deleted、**保留 vault**；用户「找回」→ 重组 latest（或选定版）写回原路径或导出。
- **跨版本搜索**：搜索增加「含历史版本」开关；命中 `version_pages_fts` → 结果定位到「文档 · 版本 · 页」，可预览该历史版本页 / 一键导出该版。

## 8. 模块与契约（新增）

```
src/pptx_finder/versioning/
  models.py     ManagedDoc / Version / Snapshot 数据类
  vault.py      snapshot(path)->version_id  /  rebuild(doc_id,version_id)->pptx字节
                list_versions(doc_id)  /  restore(...)  /  export(...)
  watcher.py    VaultWatcher(roots)：watchdog + 防抖，回调 on_saved(path)
  manager.py    纳管/取消/离线补扫/会话折叠/配额清理 编排（后台线程）
src/pptx_finder/ui/
  version_panel.py    版本时间线（缩略图/预览/恢复/导出）
  settings_dialog.py  设置面板（风格/热键/自启/受管目录/配额）
```

集成点：`app.py` 启动时拉起 `manager`（含 watcher）；主窗为受管文档挂 `version_panel`；顶栏加「设置」入口。

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| 重组 pptx 打不开（结构没还原好） | MVP 先用完整拷贝（绝对可靠）；去重阶段重组后**内部自检能否打开**，不通过则回退完整存（内部保障，非用户功能） |
| 版本库占磁盘 | 按页去重 + 自动配额清理 + 可主动删除某文档版本库 |
| 恢复时文件被 PowerPoint 占用 | 默认**导出新文件**不覆盖；要覆盖则提示先关闭 |
| 误记（临时文件/无变化保存） | 只认显式纳管目录 + 过滤 `~$` 锁文件 + 内容指纹变了才记 |
| 不碰用户文件的底线 | vault 独立存放；纳管/找回**一律先确认**，绝不静默动原文件 |

## 10. 实施路线

- **阶段 1 · MVP（核心闭环）**：文件夹纳管 + watchdog 监听保存 + **完整快照** + 版本时间线（预览/恢复/导出）+ 离线补记 + 设置面板（含开机自启）。
  - 验收：正常 Ctrl+S 即自动留版本；桌面只一个文件；可回退/导出任意版本。
- **阶段 2 · 省空间 + 整洁**：完整拷贝 → **按页去重**存储（+重组自检）；**会话折叠** + **配额清理**；**每版缩略图**。
- **阶段 3 · 杀招**：**误删/改坏找回**（删除保留 vault + 重建入口）；**跨版本内容搜索**（历史页文本入 FTS + 搜索范围开关）。

> 严格按阶段推进，每阶段都有可用产出。阶段 1 先让用户用上「一个文件可回退」，确认对路再上去重与杀招。

## 11. 测试要求（真实执行）

- **往返一致性**：snapshot→rebuild 出的 pptx 与原文件可打开/内容一致（去重阶段重点）。
- 去重正确性：改 1 页只新增对应 part；未改 part 复用。
- 监听：模拟 PowerPoint 原子保存事件 → 防抖后恰好记 1 版；无变化保存不记。
- 离线补记：进程未运行期间改动 → 启动补记最终态。
- 折叠/配额：同会话多保存折叠；超额按策略清理。
- 找回：删除原文件 → 从 vault 重建出可打开的 pptx。
- 跨版本搜：删了某内容 → 仍能搜到含它的历史版本并定位到页。
